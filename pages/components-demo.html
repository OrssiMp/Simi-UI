Composants Full & Less

Inspiré de Flutter, Simi UI propose deux types de composants: Full (Stateful) - composants complets sans classes additionnelles, et Less (Stateless) - composants avec style de base nécessitant des classes pour les éléments.

Full Components Stateful

Composants complets qui génèrent automatiquement leur contenu et style. Aucune classe additionnelle nécessaire - tout est stylisé automatiquement.

Exemple: Form Register

<!-- Full Component - Aucune classe sur les éléments enfants --> <div class="full-component component-form form-register"> <input type="text" placeholder="Nom complet"> <input type="email" placeholder="Email"> <input type="password" placeholder="Mot de passe"> <button>S'inscrire</button> </div>

Exemple: Hero Section

<!-- Full Component Hero - Contenu généré automatiquement --> <div class="full-component component-hero"></div>

Exemple: CTA Button

<!-- Full Component CTA - Texte généré automatiquement --> <button class="full-component component-cta"></button>

Less Components Stateless

Composants avec style de base qui nécessitent des classes spécifiques pour les éléments enfants. Plus de flexibilité et de contrôle sur le contenu.

Exemple: Form Register

Créer un compte

Rejoignez notre communauté

<!-- Less Component - Classes nécessaires sur les éléments --> <div class="less-component form-component form-register"> <h2 class="form-title">Créer un compte</h2> <p class="form-subtitle">Rejoignez notre communauté</p> <input type="text" class="form-input" placeholder="Nom complet"> <input type="email" class="form-input" placeholder="Email"> <input type="password" class="form-input" placeholder="Mot de passe"> <button class="form-btn">S'inscrire</button> </div>

Exemple: Card Product

Produit Premium

$99.99

Description détaillée du produit avec toutes ses fonctionnalités.

<div class="less-component card-component card-product"> <img src="..." class="card-image" alt="Produit"> <div class="card-body"> <h3 class="card-title">Produit Premium</h3> <p class="card-price">$99.99</p> <p class="card-description">Description...</p> <button class="card-btn">Acheter</button> </div> </div>

Comparaison

Full Components (Stateful)

  • ✅ Aucune classe sur les éléments enfants
  • ✅ Contenu auto-généré
  • ✅ Rapide à implémenter
  • ✅ Cohérence garantie
  • ❌ Moins de flexibilité
  • ❌ Contenu prédéfini

Less Components (Stateless)

  • ✅ Contrôle total du contenu
  • ✅ Très flexible
  • ✅ Personnalisable
  • ✅ Réutilisable
  • ❌ Classes nécessaires
  • ❌ Plus de code HTML
